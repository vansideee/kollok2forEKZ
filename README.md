
# To-Do List REST API

**Задача:** Создание API (REST) для управления списком задач (To-Do List), включающий эндпоинты для CRUD-операций (Create, Read, Update, Delete) с использованием методов GET, POST, PUT, DELETE. Данные передаются в формате JSON. Реализовано логирование событий сервера.

### 1. Ресурсы и Эндпоинты (Endpoints)

Приложение предоставляет набор методов для управления ресурсом **Task** (Задача).

**Структура задачи:**
*   `id` (int) — Уникальный идентификатор (генерируется автоматически).
*   `title` (string) — Название задачи (обязательное поле).
*   `description` (string) — Описание задачи.
*   `status` (string) — Статус выполнения (например: "todo", "in_progress", "done").

**Реализованные методы:**

1.  **GET /tasks**
    *   Возвращает список всех задач.
    *   Код ответа: `200 OK`.

2.  **POST /tasks**
    *   Создает новую задачу.
    *   Принимает JSON: `{"title": "...", "description": "...", "status": "..."}`.
    *   Код ответа: `201 Created` (при успехе) или `400 Bad Request` (при ошибке валидации).

3.  **GET /tasks/{id}**
    *   Возвращает одну задачу по её ID.
    *   Код ответа: `200 OK` или `404 Not Found`.

4.  **PUT /tasks/{id}**
    *   Обновляет существующую задачу по ID.
    *   Код ответа: `200 OK` или `404 Not Found`.

5.  **DELETE /tasks/{id}**
    *   Удаляет задачу по ID.
    *   Код ответа: `200 OK` или `404 Not Found`.

### 2. Логирование и Работа с данными

**Хранение данных:**
*   Используется in-memory хранилище (список `list`).
*   Реализован алгоритм генерации уникального ID на основе анализа существующего списка.

**Модуль логирования (Logging):**
В проект интегрирован системный сбор метрик и событий:
*   **INFO:** Записываются события успешного запуска сервера, входящие запросы, успешное создание и удаление записей.
*   **WARNING:** Фиксируются попытки обращения к несуществующим ресурсам (404) и ошибки валидации данных (400).
*   **Формат лога:** `Время - Уровень - Сообщение`.
